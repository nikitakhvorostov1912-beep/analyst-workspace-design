<risks>
1. **pingAll на 20+ каналов = 20 одновременных HTTP запросов → может затормозить backend/MCP** — для MVP не критично, пользователь редко имеет >5 connections. **Митигация:** документируем в SUMMARY; при необходимости в Phase 3 ввести concurrency limit (p-limit или семафор).
2. **fetchConnections fall back на localStorage может рассинхронизироваться с backend** — пользователь удалил connection в backend через curl, но localStorage всё ещё знает старую. **Митигация:** при каждом успешном fetchConnections — syncMCPConnections(result) перезаписывает localStorage; fallback используется только на сетевую ошибку.
3. **Switching канала в середине streaming** — useChatStream продолжает писать в старую сессию. **Митигация:** redirect на `/` сразу обрывает stream через AbortController (cleanup в useEffect useChatStream). При unmount компонента — abort fetchChat signal.
4. **shadcn dropdown недоступен через CLI** — если pnpm dlx не работает в sandbox. **Митигация:** создаём dropdown-menu.tsx вручную с обёрткой над @radix-ui/react-dropdown-menu. Это +1 npm dep.
5. **Phase 1 Settings page рендерит connections из localStorage** — теперь источник истины = backend. UI Settings в Phase 1 был read-only, его не трогаем; добавим warning в Settings page «Для CRUD — Phase 3» (или просто оставим неизменным — это известный известный stub Phase 1).
6. **Endpoint validation `http://localhost` в HTTPS-prod** — если deployement в prod, localhost endpoints не работают. **Митигация:** MVP single-user local; для prod нужен HTTPS validation в Phase 3.
</risks>
