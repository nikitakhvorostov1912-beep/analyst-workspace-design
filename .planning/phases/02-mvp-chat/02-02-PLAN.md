- `_parse_mcp_text_content(result: dict) -> dict | None` — общий парсер: MCP может возвращать как dict напрямую, так и `{"content": [{"type":"text","text":"<json>"}]}`. Функция возвращает распакованный dict или None если parse fail.
- `_build_table(args, result_dict) -> dict | None`:
  - Ожидаем result_dict с ключами columns:list (или schema:list) и rows:list (или data:list)
  - Из columns собрать `[{name, type}]`. Если type не указан — выводить из первого row (str/int/float/bool/null) и использовать "String"/"Number"/"Boolean"/"Null"
  - total = result_dict.get("total") or len(rows)
  - meta = {"query": args.get("query"), "duration_ms": result_dict.get("duration_ms")}
- `_build_object(args, result_dict) -> dict | None`:
  - header = result_dict.get("header") or {"name": ..., "type": result_dict.get("type",""), "path": args.get("filter") or args.get("link","")}
  - attributes = result_dict.get("attributes", []) — каждый элемент проверяется на {name, type}
  - tabular_sections = result_dict.get("tabular_sections", [])
  - forms = result_dict.get("forms", [])
  - templates = result_dict.get("templates", [])
- `_build_log(args, result_dict) -> dict | None`:
  - entries = result_dict.get("entries", []). Каждый элемент: time, level, user (optional), event, comment (optional)
  - Валидация level ∈ {"Info","Warning","Error","Critical"} — если нет, дефолт "Info"
  - next_cursor = result_dict.get("next_cursor")

`build_card_from_tool_result(tool_name, args, result)`:
- result_dict = _parse_mcp_text_content(result); если None → None
- match tool_name:
  - "execute_query" → _build_table(args, result_dict)
  - "get_event_log" → _build_log(args, result_dict)
  - "get_object_by_link" → _build_object(args, result_dict)
  - "get_metadata" если args.get("sections") или args.get("filter") begin with "Документ./Справочник./..." → _build_object; иначе None (summary режим — не карточка а просто TL;DR)
  - "find_references_to_object" → _build_table(args, result_dict) с колонками [object, path, presentation]
  - остальные → None

Каждый builder при partial-result building возвращает Pydantic-валидируемый dict через `Model(**data).model_dump()` — это гарантирует что мы НЕ пошлём в SSE невалидный payload (lazy validation на границе).

6. **`useMemo` на огромном rows (1000+)** — sort может занять секунды и фризить UI. **Митигация:** в Phase 2 cap rows на client = 1000 (если total > 1000 → показывать только первые 1000 + warning «Большой результат — скачайте CSV»). Это явно документировать в TableCard.
