     ```sql
     SELECT s.id, s.title, s.channel_id, s.updated_at,
            (SELECT COUNT(*) FROM messages m WHERE m.session_id = s.id) AS message_count
     FROM sessions s
     WHERE (? IS NULL OR s.channel_id = ?)
     ORDER BY s.updated_at DESC
     ```

     - yesterday: DATE(updated_at) == DATE(now - 1 day)

<risks>
1. **fetchSessionMessages возвращает огромный JSON для длинных сессий (>500 messages)** — может фризить UI. **Митигация:** в Phase 2 не делаем pagination, просто документируем (если кейс возникнет — Phase 3 добавит limit/offset). Cap в backend SELECT LIMIT 500 + warning.
2. **Auto-title background task падает молча** — пользователь не получит title. **Митигация:** ловим Exception в asyncio task → log warning. Fallback всегда — None title в БД (UI показывает «Новый чат»).
3. **Date grouping в SQL зависит от server timezone** — если sqlite default UTC, а пользователь в +3 → группы поедут. **Митигация:** делаем группировку в Python с явным `datetime.now()` (локальное серверное время) — для single-user MVP достаточно. Документируем в SUMMARY.
4. **Refresh /sessions/{id} в момент streaming → дубли сообщений** — если useChatStream хук пересоздаётся при ремоунте, но в БД сохранения ещё нет, можем показать пустой Thread. **Митигация:** initialMessages берётся из fetchSessionMessages — даже если stream был interrupted, persistent state восстанавливается из последнего save. Гарантия: save_assistant_message происходит только на event:done (на error — НЕ сохраняем half-done assistant; user message сохраняется сразу).
5. **createSession создал empty session, юзер ничего не написал → мусорная пустая сессия** в sidebar. **Митигация:** не создаём session до первого user message. «+ Новый чат» button делает только router.push("/sessions/new") (специальный URL), на котором страница знает что session_id = null → передаст null в useChatStream → run_chat_loop в backend создаст ensure_session. **Alt решение**: «Новый чат» создаёт session сразу но при unmount/navigation away без messages — backend cleanup задание удалит. Default = first option (без cleanup taska).
6. **shadcn confirm dialog не установлен, используем window.confirm** — выглядит уродливо. **Митигация:** OK для MVP; Phase 3 добавит shadcn AlertDialog (SEC-01 уже подразумевает).
</risks>
