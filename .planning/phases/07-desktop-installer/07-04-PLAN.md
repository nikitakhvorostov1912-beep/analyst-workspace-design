<objective>
Упаковать desktop/ Electron-приложение (main.js из Plan 7.1) с встроенными ресурсами (backend.exe + frontend/ из wave 1) в один Windows NSIS installer — файл `desktop/dist/analyst-setup-v1.0.exe` (~180 MB). Installer работает per-user без прав администратора, создаёт ярлыки на Desktop и в Start Menu, поддерживает русскую локализацию мастера установки.

Purpose: Закрыть DIST-04 — без installer аналитик не сможет получить приложение единым файлом для распространения.

Output: `desktop/electron-builder.yml`, `desktop/icon.ico`, `desktop/dist/analyst-setup-v1.0.exe` (артефакт build).
</objective>

<interfaces>
<!-- electron-builder контракт -->

electron-builder (v25.x) ожидает:
- В корне Electron-проекта (desktop/): package.json с main, version, devDependencies.electron-builder
- electron-builder.yml ИЛИ secret build config в package.json — выбираем yml для читаемости
- extraResources — копируются в process.resourcesPath на packaged target (т.е. desktop/resources/ → InstallDir/resources/)
- files — content приложения (main.js, preload.js, package.json) попадает в asar

NSIS target (Windows):
- oneClick: false — wizard, не auto-install
- perMachine: false — install per-user в %LOCALAPPDATA%\Programs\<productName>\ (НЕ требует admin)
- createDesktopShortcut + createStartMenuShortcut — auto-shortcuts
- shortcutName — кастомное имя ярлыка
- installerLanguages: [ru_RU, en_US] — multi-language wizard
- language: 1049 — Russian primary
- artifactName: pattern для имени .exe (${version}, ${ext})

Pre-condition wave 3 → этот план:
- wave 1: desktop/resources/backend.exe (Plan 7.2), desktop/resources/frontend/server.js (Plan 7.3)
- wave 2: desktop/main.js, desktop/preload.js, desktop/node_modules/electron* (Plan 7.1)

Без этих файлов electron-builder упадёт с понятной ошибкой («no main found», «resources missing»).
