<risks>
1. **80% coverage gate может не достигнуться** — orchestrator/loop.py имеет много branches. Karpathy Simplicity: не писать ради coverage trivial getter-тестов.
2. **Playwright install --with-deps требует sudo на Linux** — на GitHub Actions runner есть sudo по умолчанию, локально может не быть. Mitigation: используем `--with-deps chromium` только в CI; локально разработчик ставит вручную через `pnpm exec playwright install chromium`.
3. **MSW vs route()** — мы выбрали route(). MSW добавляет browser/Node двойственность сложности. route() simple & works. Mitigation: документируем выбор; если нужны более сложные шеймы (например shared mocks между vitest и playwright) — добавим в v2.
4. **Conflict с Phase 2 vitest mocks** — useChatStream.test уже монтирует fetchChat mock; vitest и Playwright не пересекаются (vitest = unit, playwright = e2e). Безопасно.
5. **CI время выполнения** — 3 jobs параллельно, e2e ~3-5 минут. Mitigation: concurrency group cancel-in-progress; cache pnpm/pip ускоряет.
6. **safety._pending в test_orchestrator_coverage.py** — может состоянить между тестами. Mitigation: autouse fixture _reset_pending уже описан в Task 1.
</risks>
