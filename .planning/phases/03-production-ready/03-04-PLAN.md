LogCard backend endpoint — stateful подход (CONTEXT.md):

Текущая БД messages.cards JSON — это финальный snapshot. Для load-more
нужен исходный tool_call (name="get_event_log", args={...filters}) +
endpoint MCP. Решение:
1. Добавить миграцию v3 — таблица `card_states`:
   ```sql
   CREATE TABLE card_states (
     card_id TEXT PRIMARY KEY,
     session_id TEXT NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
     message_id TEXT NOT NULL,
     tool_name TEXT NOT NULL,
     original_args TEXT NOT NULL,  -- JSON
     channel_id TEXT NOT NULL,
     created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
   )
   ```
2. При построении карточки log в orchestrator/cards.py — также сохранять
   card_state, генерируя card_id (UUID4) и помещая его в card.payload.
3. LogCard.payload type расширяется: `card_id?: string` (optional для
   обратной совместимости с уже сохранёнными).
4. Endpoint POST /sessions/{sid}/messages/{mid}/cards/{cid}/load-more:
   - Body: {cursor: string}
   - Lookup card_states by card_id (cid); если не найдено → 404.
   - Lookup endpoint через channel_id → lookup_mcp_endpoint.
   - mcp = MCPClient(endpoint); initialize; call_tool("get_event_log", {**original_args, cursor})
   - Парсим результат как LogCard (используем существующий _build_log_card).
   - Возвращаем {entries, next_cursor}.

curl-builder:
```ts
export function buildCurlCommand(
  toolCall: ToolCallRecord,
  mcpEndpoint: string,
  mcpSessionId?: string,
): string {
  const body = {
    jsonrpc: "2.0",
    id: 1,
    method: "tools/call",
    params: { name: toolCall.name, arguments: toolCall.args },
  };
  const headers = [
    "-H 'Content-Type: application/json'",
    "-H 'Accept: application/json, text/event-stream'",
  ];
  if (mcpSessionId) headers.push(`-H 'Mcp-Session-Id: ${mcpSessionId}'`);
  // Экранирование single quotes в JSON
  const jsonStr = JSON.stringify(body).replace(/'/g, "'\\''");
  return `curl -X POST '${mcpEndpoint}' \\\n  ${headers.join(" \\\n  ")} \\\n  -d '${jsonStr}'`;
}
```

mcpEndpoint в frontend нужно получить — он не сохраняется в tool_call.
Решение: 
- В props ToolTrace добавить `mcpEndpoint?: string` (если null/undefined — placeholder `<MCP_ENDPOINT>` в curl).
- В AssistantMessage брать active connection's endpoint из localStorage (getMCPConnections + activeChannelId) и пробрасывать.

       async def load_more(sid, mid, cid, body: LoadMoreRequest, db=Depends(get_db)):
           state = await get_card_state(db, cid)
           if state is None or state["session_id"] != sid or state["message_id"] != mid:

           endpoint = await lookup_mcp_endpoint(db, state["channel_id"])
           if endpoint is None:

           args = {**json.loads(state["original_args"]), "cursor": body.cursor}
           try:
               async with MCPClient(endpoint) as mcp:
                   await mcp.initialize()
                   result = await mcp.call_tool(state["tool_name"], args)

           # парсим entries + next_cursor — используем _build_log_card или прямой парс
           card = build_card_from_tool_result(state["tool_name"], args, result)
           if card is None or card["type"] != "log":

           return LogPagePayload(entries=card["payload"]["entries"], next_cursor=card["payload"].get("next_cursor"))

<risks>
1. **navigator.clipboard.writeText недоступен в insecure context** (http без HTTPS, кроме localhost) — в production за reverse-proxy с HTTPS работает; в чисто HTTP deployment fallback нужен. Mitigation: catch error → publishToast «Не удалось скопировать. Используйте HTTPS или копируйте вручную». Уже в плане.
2. **CSP может блокировать clipboard** — Plan 3.2 CSP не упоминает clipboard restriction; navigator.clipboard работает с тем же origin, не нужны `connect-src` исключения. Безопасно.
3. **card_id не существует у legacy cards** (созданных Phase 2 до миграции v3) — load-more кнопка disabled когда `card.payload.card_id` undefined. LogCard.tsx уже имеет fallback (Phase 2 placeholder).
4. **Migration v3 на существующей prod БД** — добавление новой таблицы безопасно (нет данных к копированию); ALTER TABLE не используется. Backward compat 100%.
5. **mcpEndpoint в frontend prop chain** — sessionId/messageId надо пробросить из page до AssistantMessage. Это 3 уровня props. Mitigation: пробрасываем явно (драйв context API не нужен).
6. **Дублирование load-more state** — LogCard локально хранит entries, и LogCard.payload.entries — снапшот. После reload страницы load-more entries исчезают. Mitigation: документируем как known limitation (refresh = новая страница только); persisting load-more entries в DB — v2.
</risks>
