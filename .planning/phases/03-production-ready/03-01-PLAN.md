## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM provider → backend orchestrator | Внешний ответ может быть 429/4xx/5xx или вернуть error JSON в SSE; контролируется через mapping в loop.py |
| MCP server (локальный 1С EPF) → backend orchestrator | initialize/list_tools/call_tool могут падать (process crash, network); должно деградировать в SSE-error, не 500 |
| Backend SSE → browser useChatStream | Browser должен корректно роутить error.code → не показывать stack trace |
| Browser → backend /connections/{id}/ping | Retry button дёргает endpoint; нет auth (MVP single-user — Phase 3 явно не вводит auth) |

<risks>
1. **respx mock для httpx streaming 429** — может оказаться сложным мокать chunk-by-chunk. Mitigation: использовать monkeypatch на app.orchestrator.loop.LLMClient.stream_chat_completion и возвращать AsyncIterator, который сразу бросает LLMRateLimitError (минуя реальный httpx). Тест check exception path в loop.py, не клиент.
2. **fake-timers + countdown** — vi.useFakeTimers с setInterval может конфликтовать с React useEffect cleanup. Mitigation: использовать act() обёртку, рассмотреть `vi.useFakeTimers({shouldAdvanceTime:true})`.
3. **Banner pинопадание на CSP** — после Plan 3.2 CSP может заблокировать inline стили. Mitigation: ConnectionStatusBanner использует Tailwind classes, не inline-style → совместимо с строгой CSP.
4. **Backwards compat error code** — Plan 2.x фронта мог рендерить code llm_4xx/llm_5xx/mcp_connect_error. Mitigation: оба переименования делаются атомарно в этом плане; мест использования всего 2 (useChatStream + 0 tests Phase 2 явно проверяют код). Безопасно.
5. **retry button race** — пользователь жмёт Повторить дважды. Mitigation: bannerRetrying state блокирует кнопку при первом клике.
</risks>
