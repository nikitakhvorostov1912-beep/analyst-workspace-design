1. **UI toggle** в Header — глобальный (не per-session) флаг, persist в localStorage `analyst.anon_enabled`

Зеркало текущих контрактов backend → frontend:

backend/app/clients/mcp.py:
```python
class MCPClient:
    async def call_tool(self, name: str, arguments: dict) -> dict
```
Расширяем сигнатуру: `call_tool(name, arguments, anon_enabled: bool = False)`. Anon флаг кладётся либо в HTTP заголовок (`X-Anon-Enabled: true`), либо в arguments (`_anon: true`) — выбор делает план, ориентир: capability-map описывает submit_for_deanonymization как отдельный tool, значит anon mode активируется через MCP server config / per-session, а не per-call. Поэтому: header `X-Anon-Enabled` отправляется в `_post` для всех вызовов tools/call в этой сессии. MCP server интерпретирует — отдаёт анонимизированные значения.

backend/app/orchestrator/loop.py:
```python
async def run_chat_loop(db, request, x_llm_api_key, llm_endpoint, llm_model) -> AsyncIterator[str]
```
Расширяем сигнатуру: добавляем `x_anon_enabled: bool = False` параметр. Прокидываем в `MCPClient(endpoint, headers={"X-Anon-Enabled": "true"} if anon else {})`.

backend/app/routes/log_cards.py:
```python
@router.post("/sessions/{sid}/messages/{mid}/cards/{cid}/load-more")
```
Добавляем второй endpoint:
```python
@router.post("/sessions/{sid}/messages/{mid}/cards/{cid}/deanonymize")
async def deanonymize_card(sid: str, mid: str, cid: str, request: DeanonymizeRequest, db: ...) -> DeanonymizeResponse
```

DeanonymizeRequest:
- tokens: list[str] (min_length=1, max_length=200, каждый matches r'\[[A-Z]+-\d+\]')

DeanonymizeResponse:
- mapping: dict[str, str]   # {"[ORG-001]": "ООО Ромашка", ...}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /chat | X-Anon-Enabled header; untrusted (юзер может установить вручную, но это его выбор) |
| Browser → /deanonymize | tokens list; untrusted, валидируется regex strict |
| /deanonymize → MCP submit_for_deanonymization | trusted, MCP server решает что отдать |
| MCP response → Frontend (через /deanonymize) | trusted после Pydantic strict; реальные значения **никогда** не сохраняются в DB или localStorage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | /deanonymize tokens param | mitigate | Pydantic strict + regex `^\[[A-Z]+-\d+\]$` для каждого token + max_length=200 |
| T-04-02 | Information Disclosure | Real values utech in client-side | mitigate | Real values хранятся только в React state (revealedMap), не в localStorage, не в DB. После reload — токены снова видны, нужно раскрывать заново. |
| T-04-03 | Information Disclosure | Real values в ToolTrace JSON | accept | Trace показывает оригинальный tool_result от MCP (с токенами); раскрытие НЕ применяется к Trace. Документировано в SUMMARY. |
| T-04-04 | Spoofing | X-Anon-Enabled header можно подделать | accept | Single-user MVP, юзер сам ставит флаг. MCP server решает доверять или нет (там per-account anon mode). |
| T-04-05 | Repudiation | Действие «раскрыть» не audit-логировано | accept | MVP. Production v2 — log to caplog/event_log. |
| T-04-06 | DoS | Множественные POST /deanonymize | accept | MVP single-user; MCP server должен иметь свой rate limit |
| T-04-07 | Information Disclosure | Реальные значения в HTTP cache | mitigate | Response /deanonymize включает `Cache-Control: no-store, private` |
| T-04-08 | Tampering | card_id forge | mitigate | Ownership check: state.session_id == sid AND state.message_id == mid (как в load-more) |

<risks>
1. **MCP submit_for_deanonymization shape неизвестен** — capability-map описывает имя, но точную форму mapping мы делаем по соглашению `{mapping: {"[ORG-001]": "real"}}`. Реальная 1С MCP может вернуть другой формат. Mitigation: в `routes/log_cards.py:deanonymize_card` несколько fallback парсеров (mapping / map / replacements ключи). SUMMARY фиксирует реальный формат после первого E2E теста.

2. **Markdown highlightAnonTokens может ломать markdown структуру** — если применить text replacement внутри `<a href>` URL то URL рассыпется. Mitigation: применяем только в renderers `p/td/li/code(inline)`, НЕ в `a`. Тестировать на сообщении с ссылкой содержащей [ORG-001].

3. **localStorage не доступен в SSR** — Next.js SSR на /page.tsx может падать при getAnonEnabled() во время initial render. Mitigation: useState(false) + useEffect для чтения localStorage on mount.

4. **ALTER TABLE при наличии данных** — если кто-то запустил миграцию v3, накопил card_states, мы добавляем колонку. SQLite ADD COLUMN без NOT NULL и без DEFAULT работает. Verify в test_migration_v4.

5. **Производительность extractAnonTokens на больших payload** — TableCard может содержать 1000 строк. Regex on stringified JSON ~10ms — приемлемо. Если станет проблемой — debounce.

6. **HTTP cache real values** — Cache-Control: no-store обязателен. Mitigation: явно в response headers endpoint.

7. **Race condition: пользователь жмёт «Раскрыть» дважды быстро** — два параллельных запроса. Mitigation: disable button во время pending state.
</risks>
