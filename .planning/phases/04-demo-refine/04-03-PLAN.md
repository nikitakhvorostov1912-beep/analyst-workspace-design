DDL v5:
```sql
CREATE VIRTUAL TABLE IF NOT EXISTS messages_fts USING fts5(
    content,
    session_id UNINDEXED,
    message_id UNINDEXED,
    tokenize = 'porter unicode61'
);

CREATE TRIGGER IF NOT EXISTS messages_ai AFTER INSERT ON messages BEGIN
    INSERT INTO messages_fts(rowid, content, session_id, message_id)
    VALUES (new.rowid, COALESCE(new.content, ''), new.session_id, new.id);
END;

CREATE TRIGGER IF NOT EXISTS messages_au AFTER UPDATE OF content ON messages BEGIN
    UPDATE messages_fts SET content = COALESCE(new.content, '') WHERE rowid = old.rowid;
END;

CREATE TRIGGER IF NOT EXISTS messages_ad AFTER DELETE ON messages BEGIN
    DELETE FROM messages_fts WHERE rowid = old.rowid;
END;

CREATE TABLE IF NOT EXISTS metadata_cache (
    channel_id TEXT NOT NULL,
    object_path TEXT NOT NULL,
    object_type TEXT NOT NULL,
    name TEXT NOT NULL,
    presentation TEXT,
    fetched_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (channel_id, object_path)
);

CREATE INDEX IF NOT EXISTS idx_metadata_cache_channel_name ON metadata_cache(channel_id, name);
```

Backfill: после создания FTS5 — `INSERT INTO messages_fts SELECT rowid, content, session_id, id FROM messages` (idempotent при использовании fts5 vocab MERGE; для миграции мы делаем чистый INSERT — выполняется только при первом проходе v5; на повторных версия не меняется).

SQL:
```sql
SELECT
    f.session_id, s.title AS session_title, f.message_id,
    snippet(messages_fts, 0, '<mark>', '</mark>', '...', 32) AS snippet,
    m.created_at, s.channel_id
FROM messages_fts f
JOIN messages m ON m.id = f.message_id
JOIN sessions s ON s.id = f.session_id
WHERE messages_fts MATCH ?
  {AND s.channel_id = ?}
ORDER BY rank
LIMIT ?
```

backend/app/routes/connections.py — добавить:
```python
@router.get("/channels/{channel_id}/metadata-suggest")
async def metadata_suggest(channel_id: str, q: str = Query(...,min_length=1,max_length=80), limit: int = 20, db: ...) -> MetadataSuggestResponse
```

Behavior:
1. Lookup channel в `mcp_connections` (404 если нет)
2. Query cache: `SELECT * FROM metadata_cache WHERE channel_id = ? AND name LIKE ?` (prefix match `q%` + middle match `%q%`)
3. Если cache rows есть И самая старая запись < TTL_SECONDS (3600) → возвращаем
4. Иначе trigger refresh: создаём MCPClient, initialize, вызываем `get_metadata({"detail": False})` (legkij overview), парсим objects, INSERT OR REPLACE в metadata_cache. После refresh — повторяем SELECT.
5. Если MCP недоступен И в cache что-то есть → возвращаем stale (с флагом `stale: true`)

```python
class MetadataSuggestItem(BaseModel):

    name: str              # "ОПП"
    full_path: str         # "Документ.ОПП"
    presentation: str | None

class MetadataSuggestResponse(BaseModel):
    items: list[MetadataSuggestItem]
    cached: bool
    stale: bool = False
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /search?q= | untrusted query string, FTS5 экранируется |
| Browser → /channels/{id}/metadata-suggest | untrusted q, prefix filter SQL parameterized |
| /search snippet → frontend `<mark>` HTML | trusted после sanitize (только `<mark>` теги whitelisted) |
| metadata_cache → MCPClient refresh | trusted (single-user, our own MCP) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-14 | Injection | /search FTS5 query | mitigate | `_fts5_safe_query` экранирует quote и spec chars. Параметризованный SQL для остальных bindings. |
| T-04-15 | Injection | /metadata-suggest q | mitigate | SQL parameter binding (?), без construction string. |
| T-04-16 | DoS | /search с очень дорогим query | mitigate | min_length=2 (не один char), limit ≤ 50; FTS5 имеет встроенный rank ограничение |
| T-04-17 | DoS | /metadata-suggest без cache → MCP storm | mitigate | TTL-кеш предотвращает множественные refresh в одной минуте; параллельные cache-miss обработают только один refresh (atomic SELECT MAX + UPDATE) — accept race condition (на N запросов получаем N refresh — допустимо для single-user) |
| T-04-18 | XSS | snippet `<mark>` в HTML | mitigate | escape всех HTML chars в snippet КРОМЕ `<mark>` и `</mark>` тегов через regex replace на frontend. Тест покрывает |
| T-04-19 | Information Disclosure | /search возвращает messages из других channel | mitigate | Если channel параметр передан — WHERE s.channel_id = ?; если нет — все channels пользователя (single-user OK) |
| T-04-20 | Tampering | LLM приходит response с command injection в @-mention text | accept | mention это plain text в prompt, LLM может игнорировать; не выполняется как код |
