**1. `MCPConnectionForm.tsx`** — props: `{ initial?: MCPConnection | null; onSaved: (conn: MCPConnection) => void; onCancel?: () => void }`.

Fields (state): `name`, `endpoint`, `channel`, `anon_enabled`. При `initial` non-null — pre-fill полей.

Кнопки в ряд:

- **«Отмена»** — `onCancel?.()` (только в edit-mode).

**2. `MCPConnectionList.tsx`** — props: `{ initialConnections: MCPConnection[]; onChanged?: () => void }`. State: `connections`, `editingId | null`, `addingNew: bool`, `deleteId | null`.

Рендер:
- Если `connections.length === 0` — текст «Подключения не настроены».
- Иначе — список row-ами: `name (channel) — endpoint — статус dot (если есть last_seen_at, мы не пингуем здесь, статус «—»; ping — отдельная кнопка)`.
- В каждом row кнопки: `[Тест]` (вызывает `pingConnection(id)`, показывает toast результат), `[Изменить]` (`setEditingId(conn.id)`), `[Удалить]` (`setDeleteId(conn.id)`).
- Кнопка `+ Добавить подключение` внизу.
- Если `editingId === conn.id` → inline рендерится `<MCPConnectionForm initial={conn} onSaved={() => { reload(); setEditingId(null) }} onCancel={() => setEditingId(null)} />`.
- Если `addingNew` → `<MCPConnectionForm onSaved={() => { reload(); setAddingNew(false) }} onCancel={() => setAddingNew(false)} />`.
- Confirm dialog: `<AlertDialog open={deleteId !== null}>` — текст «Удалить подключение `{name}`? Это действие нельзя отменить.» Кнопки: Отмена, Удалить (на Удалить → `deleteConnection(deleteId); toast; reload(); setDeleteId(null)`).
- `reload()` = `fetchConnections().then(setConnections).then(onChanged?.())`.

**3. `LLMConfigForm.tsx`** — props: `{ initial: LLMConfigResponse | null; onSaved?: () => void }`. State: `endpoint`, `model`, `temperature` (number, default 0.3), `api_key` (string, prefill from `getLLMApiKey()`), `errors`, `hasExisting = initial !== null`.

UI:

Логика:

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | I | api_key в sessionStorage | accept | Trade-off: sessionStorage > React state в плане UX (выживает refresh), хуже React state в плане XSS exposure. CSP уже включён (Phase 3 SEC-02), XSS-vectors минимизированы. Документировать в USER.md (Plan 5.5) |
| T-05-07 | T | XSS через имя connection / endpoint | mitigate | React по умолчанию экранирует строки в JSX, мы не используем dangerouslySetInnerHTML в этих компонентах |
| T-05-08 | E | DELETE без confirm | mitigate | AlertDialog обязателен для всех DELETE кнопок |
| T-05-09 | I | api_key в network logs | mitigate | api_key передаётся только в Authorization-style header `X-LLM-API-Key` через HTTPS-ready backend; не логируется бэкендом |
