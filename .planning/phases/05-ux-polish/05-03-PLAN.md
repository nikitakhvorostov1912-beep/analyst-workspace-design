State:

- `createdConnection: MCPConnection | null` (заполняется по onSaved MCPConnectionForm + успешный ping)

    - Кнопки footer: `[Пропустить]` (slate, левый край) + `[Далее →]` (primary, disabled пока !pingPassed)

    - h2 «Настройте LLM»

**2. Обновить `frontend/app/page.tsx`:**

Добавить state `showOnboarding: bool | null` (null = ещё не определили).

Заменить mount useEffect:
```
useEffect(() => {
  let cancelled = false;
  (async () => {
    const flag = getOnboardingCompleted();
    if (flag) { setShowOnboarding(false); /* continue с обычной загрузкой */ }
    else {
      try {
        const [conns, llm] = await Promise.all([fetchConnections(), fetchLLMConfig()]);
        if (cancelled) return;
        const hasBoth = conns.length > 0 && llm !== null;
        if (hasBoth) {
          // Legacy users: уже всё настроено
          setOnboardingCompleted(true);
          setShowOnboarding(false);
        } else {
          setShowOnboarding(true);
        }
      } catch {
        // Backend недоступен — не показываем onboarding, показываем error state
        setShowOnboarding(false);
      }
    }
    // ВСЕГДА: загрузить начальный state (connections + active channel) для последующего рендера
    const conns = await fetchConnections().catch(() => []);
    if (cancelled) return;
    setHasConfig(conns.length > 0); // временно — Plan 5.4 переделает hasConfig полностью
    setLocalActiveChannelId(getActiveChannelId());
    setReady(true);
  })();
  return () => { cancelled = true; };
}, []);
```

Между ранним return-блоком `if (!ready)` и `if (!hasConfig)` добавить:
```
if (showOnboarding) {
  return (
    <>
      <OnboardingDialog
        open={true}
        onComplete={(chId) => { setShowOnboarding(false); if (chId) setActiveChannelId(chId); /* re-fetch */ }}
        onSkip={() => setShowOnboarding(false)}
      />
      <BackendIndicator />
    </>
  );
}
```

После закрытия OnboardingDialog нужно повторно загрузить `connections + llm` и перерисовать. Реализовать через `setShowOnboarding(false)` + `window.location.reload()` ИЛИ через явный refetch (паттерн второго мерge — без reload, чище). Реализовать без reload — в onComplete вызвать локальный `refresh()` helper, который повторно делает fetchConnections / setHasConfig.

ВАЖНО: этот Plan **не удаляет** `getMCPConnections()/getLLMConfig()` вызовы — это работа Plan 5.4. Сейчас оставляем существующую `hasConfig` логику как есть (она работает с localStorage), но добавляем onboarding путь до того как hasConfig проверится. Дублирование терпимо до Plan 5.4.
