# Сценарий демо — 1С Аналитик (15 минут)

**Версия:** Phase 4 (2026-05)
**Аудитория:** 1 аналитик + 1 наблюдатель
**Цель:** Получить честный фидбек от аналитика на реальных сценариях. НЕ продать инструмент.

> Подготовка наблюдателя: открыть [docs/DEMO-OBSERVER-CHECKLIST.md](./DEMO-OBSERVER-CHECKLIST.md).
> После демо: аналитик заполняет [docs/DEMO-FEEDBACK-TEMPLATE.md](./DEMO-FEEDBACK-TEMPLATE.md).
> Руководство аналитика полностью: [docs/USER.md](./USER.md).

---

## Подготовка (≤ 10 минут до встречи)

**Фасилитатор делает до встречи:**

1. Запустить стек:
   ```bash
   docker compose up
   # или вручную: uvicorn + pnpm dev
   ```
2. Проверить http://localhost:3010 — фронтенд открывается.
3. Проверить Настройки → Подключения — два MCP-подключения («РТ» и «УСО» или аналогичные) уже добавлены.
4. Проверить Настройки → LLM — API-ключ уже сохранён.
5. (Опционально — для preview без живой 1С) Загрузить демо-данные:
   ```bash
   python scripts/seed-demo-data.py --clean
   # Создаёт 5 демо-сессий с примерами всех 6 типов карточек
   ```
   Если используете seed-данные — аналитик увидит статичные cards (не live 1С).
   Для **реального демо на живой 1С** seed не нужен.
6. Прокинуть URL аналитику (localhost:3010 или ngrok-туннель).
7. Cmd+K — убедиться что CommandPalette открывается (тест работоспособности).

---

## Раздел 0: Введение (1 мин)

**Что говорить аналитику:**

> «Это чат-консоль для работы с базами 1С на естественном языке. Вы пишете вопрос по-русски —
> система сама решает какие инструменты вызвать, запрашивает данные и возвращает ответ
> с таблицей или другой карточкой. Сегодня меня не интересует "вам понравилось?" —
> меня интересуют конкретные вещи которые мешали и чего не хватало.»

**НЕ говорить:** «Это очень умная система», «LLM под капотом», «MCP».

**Передать управление клавиатурой аналитику.**

---

## Раздел 1: Подключение (2 мин)

**Действия аналитика:**

1. Открыть Настройки (шестерёнка в шапке).
2. Вкладка **Подключения** — видны 2 уже добавленных подключения к базам.
3. Нажать **Ping** на каждом подключении.
   - **Ожидаемо:** зелёная отметка «ok» + время ответа в мс.
4. Вкладка **LLM** — API-ключ уже заполнен. Нажать **Тест**.
   - **Ожидаемо:** «ok» — LLM completion работает.
5. Закрыть Настройки → вернуться на главный экран.

**Что наблюдатель фиксирует:**
- Нашёл ли аналитик Settings самостоятельно?
- Понял ли разницу вкладок «Подключения» и «LLM»?

**Резервный сценарий — Ping красный:**
> «Это значит MCP Toolkit не запущен — видите красный баннер вверху? Покажу как выглядит когда всё работает.»
> Переключить на seed-сессию и продолжить.

---

## Раздел 2: Первый prompt — простой обзор (3 мин)

**Channel selector:** убедиться что выбрана база «РТ» (или первая доступная).

**Действия:**

1. В поле ввода нажать на чип **«Обзор базы»** (QuickPrompt chip внизу чата).
   - Отправляет: «Обзор базы — перечисли подсистемы, основные документы»
2. **Наблюдать streaming:** в строке статуса над полем ввода появляется «Анализирую...»
   → «Вызываю get_metadata...» → «Формирую ответ...»
3. **Ожидаемый ответ:** ≤30 секунд. TL;DR текстом + TableCard или ObjectCard с перечнем подсистем.

**Показать trace:**

4. Нажать раскрыть **▶ Trace** под ответом.
   - Видно: 1 tool call (get_metadata), аргументы JSON, duration_ms.

**Вопрос аналитику:** «Что вы ожидали увидеть первым в этом ответе?»

**Резервный сценарий — ответ слишком длинный / нечитаемый:**
> «Это хорошее наблюдение. Запишу. Продолжим следующий шаг.»

---

## Раздел 3: Сложный prompt — журнал и таблица с cards (4 мин)

### Часть 3а: Журнал регистрации (2 мин)

1. В поле ввода набрать `/` — появляется **SlashCommand popover** со списком команд.
2. Выбрать `/journal` или набрать полностью:
   ```
   /journal Период=Час, Уровень=Error+Warning
   ```
   Нажать Enter (отправить).
3. **Ожидаемый ответ:** LogCard с цветными строками — Error красный, Warning жёлтый, Info серый.
4. Если card показывает «Загрузить ещё» (кнопка внизу) — нажать.
   - **Ожидаемо:** следующие записи догружаются инкрементально (cursor-fetch).

### Часть 3б: Таблица с документами (2 мин)

5. Набрать новый prompt:
   ```
   Покажи документы реализации за вчера, group by контрагент
   ```
   Отправить.
6. **Ожидаемый ответ:** TableCard с колонками «Контрагент», «Документ», «Сумма».
7. Нажать заголовок колонки «Сумма» — сортировка по убыванию.
8. Нажать **«Экспорт CSV»** (кнопка внизу TableCard).
   - **Ожидаемо:** браузер скачивает .csv файл.
9. (Если LLM вернул агрегированный итог) Может появиться **MetricCard** со спарклайн-графиком — показать аналитику.

**Вопрос аналитику:** «Как вы обычно смотрите на этот журнал — через 1С или выгрузкой?»

---

## Раздел 4: Анонимизация (2 мин)

**Контекст:** при работе с чужими клиентами не хочется передавать реальные имена контрагентов в LLM.

1. Нажать иконку **Анонимизация** в шапке (замок/значок).
   - Появляется **amber-полоса** вверху чата «Анонимизация включена».
2. Набрать:
   ```
   Найди контрагентов с задолженностью более 100к
   ```
   Отправить.
3. **Ожидаемый ответ:** в тексте и TableCard вместо реальных имён — токены `[ORG-001]`, `[INN-001]`, `[ORG-002]`.
4. В футере карточки нажать **«Раскрыть реальные значения»**.
   - **Ожидаемо:** токены заменяются реальными именами inline, появляется badge «Реальные значения».
5. Выключить анонимизацию (повторный клик на иконку в шапке).

**Вопрос аналитику:** «Вы работаете с несколькими клиентами? Нужна ли такая опция в типичном сценарии?»

---

## Раздел 5: Channel switching (2 мин)

1. В шапке нажать **Channel selector** (dropdown с именем текущей базы).
2. Выбрать вторую базу («УСО» или другую).
3. Наблюдать: sidebar может показать новые или пустые сессии для этой базы.
4. Набрать:
   ```
   /find Контрагент
   ```
   Отправить.
5. **Ожидаемый ответ:** **ReferencesCard** — сгруппировано по типам: Реквизит / Подчинённый / Шаблон / Право.
6. Нажать на строку из группы «Реквизит» — отправляется следующий prompt «Покажи Документ.ОПП».

**Что наблюдатель фиксирует:**
- Понял ли аналитик что переключился на другую базу?
- Ожидал ли другой вид результата?

---

## Раздел 6: Trace + Copy as curl + Cmd-K (2 мин)

### Trace + curl (1 мин)

1. Раскрыть **▶ Trace** последнего ответа.
2. Нажать **«Скопировать как curl»** — toast «Скопировано в буфер».
3. Показать (вставить в текстовый редактор) — это воспроизводимый HTTP-запрос к backend.

**Зачем:** аналитик видит что именно ушло в систему. Прозрачность.

### Cmd-K / CommandPalette (1 мин)

4. Нажать **Cmd+K** (Mac) или **Ctrl+K** (Windows).
   - Открывается **CommandPalette** — поиск по истории сессий.
5. Набрать «реализац» — появляются результаты с snippet из прошлых сессий.
6. Нажать на результат — переход к сессии, scroll к нужному сообщению.

---

## Раздел 7: Wrap-up (1 мин)

**Что говорить:**

> «Это всё что я хотел показать. Спасибо. Сейчас передам вам шаблон обратной связи.
> Пожалуйста, заполните его за 10 минут — у нас только 3 поля. Дедлайн — завтра в 12:00.»

**Передать:** [docs/DEMO-FEEDBACK-TEMPLATE.md](./DEMO-FEEDBACK-TEMPLATE.md)

**Согласовать:**
- Дедлайн для заполнения: 24 часа
- Канал для отправки: Telegram / email / GitHub issue

## После демо (фасилитатор)

1. Прочитать заполненный DEMO-FEEDBACK-TEMPLATE.md от аналитика.
2. Заметки наблюдателя из DEMO-OBSERVER-CHECKLIST.md.
3. Добавить пункты в `.planning/BACKLOG-POST-MVP.md` по категориям (UX/Performance/Cards...).
4. Пересмотреть этот DEMO-SCRIPT.md если какие-то шаги не совпали с реальностью.

> **Важно:** DEMO-SCRIPT.md отражает состояние Phase 4. Если с момента последнего
> обновления прошло >14 дней — проверить названия tools/cards на актуальность.

## Резервные сценарии (Quick Reference)

| Ситуация | Действие |
|----------|---------|
| MCP не отвечает | Показать красный баннер ConnectionStatusBanner + кнопку Retry. «Это выглядит так когда EPF не запущен.» |
| LLM rate limit | Toast с обратным отсчётом `retry-after`. Подождать или переключить на другую модель в Settings. |
| Confirm dialog при `/sql DROP...` | Показать — аналитик видит safety net. «Это специально для опасных команд.» |
| LLM вернул пустой ответ | «Попробуем переформулировать.» + набрать более конкретный prompt. |
| Аналитик задаёт вопрос вне сценария | Позволить задать (это ценный фидбек!). Наблюдатель фиксирует запрос. |
| Аналитик хочет скопировать результат | Показать Экспорт CSV или Copy as curl. |
| Build сессии не нашлась через Cmd-K | «FTS индексирует только сохранённые сессии.» Переключить на уже существующую сессию из sidebar. |

## Тайминг (суммарно ≤ 15 мин)

| Раздел | Время | Ключевой момент |
|--------|-------|----------------|
| 0. Введение | 1 мин | Цель — фидбек, не продажа |
| 1. Подключение | 2 мин | Ping зелёный |
| 2. Первый prompt | 3 мин | Streaming виден аналитику |
| 3. Журнал + таблица | 4 мин | LogCard + TableCard + CSV |
| 4. Анонимизация | 2 мин | Токены → раскрытие |
| 5. Channel switch | 2 мин | ReferencesCard |
| 6. Trace + Cmd-K | 2 мин | Curl + поиск |
| 7. Wrap-up | 1 мин | Дедлайн для FEEDBACK |
| **Итого** | **≤17 мин** | Буфер 2 мин на вопросы |
