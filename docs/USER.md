# Руководство аналитика — 1С Аналитик

Это руководство для бизнес-аналитика, который впервые запускает 1С Аналитик.
Вам не нужно знать SQL, BSL или MCP — LLM всё делает за вас.

---

## Что это

**1С Аналитик** — чат-консоль для работы с базами 1С на естественном языке.
Вы пишете вопрос по-русски, система сама выбирает нужные инструменты, запрашивает данные
и возвращает ответ с inline-карточкой (таблица, объект, журнал).

**Ничего не устанавливается на 1С** — используется штатный EPF «MCP Toolkit», который запускается
в 1С и слушает на localhost.

---

## Подключаем 1С

### Шаг 1. Запустить MCP Toolkit

Откройте базу 1С → Меню Сервис → Файл → Открыть → выберите `APIСигма.epf`.
В форме нажмите **Запустить** (встроенный сервер). Порт по умолчанию: **6010**.
После успешного старта в строке статуса появится «Сервер запущен».

### Шаг 2. Добавить подключение в 1С Аналитик

1. Открыть http://localhost:3010
2. Кнопка **Настройки** (шестерёнка в шапке) → вкладка **Подключения**
3. Нажать **+ Добавить**
4. Заполнить:
   - **URL**: `http://localhost:6010/mcp`
   - **Имя**: любое, например «Русский Транзит»
5. Нажать **Ping** — должно появиться «ok» со временем ответа в мс
6. Сохранить

После сохранения подключение появится в выпадающем меню **Channel selector** в шапке.

---

## Подключаем LLM

1. Настройки → вкладка **LLM**
2. Заполнить:
   - **Endpoint**: `https://api.openai.com/v1` (или другой OpenAI-совместимый)
   - **API ключ**: ваш ключ (sk-...)
   - **Модель**: `gpt-4o` или `mimo-32b` или другое название модели
   - **Temperature**: 0.3 рекомендуется
3. Нажать **Тест** — если всё верно, появится «ok»

API ключ хранится только в вашем браузере (localStorage), на сервер не сохраняется.

---

## Задаём первый вопрос

1. Убедитесь что в Channel selector выбрано нужное подключение
2. Введите вопрос в поле ввода внизу. Например:
   - «Покажи первые 10 документов реализации за апрель»
   - «Что есть в справочнике Контрагенты для ИНН 7701234567?»
   - «Покажи журнал ошибок за последние 2 часа»
3. Нажмите **Отправить** (Enter или Cmd+Enter)
4. Наблюдайте статус: Анализирую → Вызываю tool → Формирую ответ
5. Ответ появится с inline-карточкой: таблица, объект или журнал

---

## Читаем trace вызовов инструментов

После каждого ответа ассистента внизу есть ссылка вида **▶ 2 инструмента, 843 мс**.
Кликните на неё — раскроется список вызовов инструментов с аргументами и результатами.

Это полезно чтобы понять что именно LLM запросил у 1С, и убедиться что данные верные.

---

## Копируем как curl

В каждой строке trace есть кнопка **Скопировать как curl**.
Нажмите — команда попадает в буфер обмена. Вставьте в терминал чтобы воспроизвести
вызов инструмента напрямую.

Подробнее о формате: [docs/CURL.md](CURL.md).

## Подтверждение execute_code

Если LLM решает выполнить код 1С (`execute_code`) с опасными операциями
(запись, удаление, транзакции), появится **диалог подтверждения**.

Вы увидите:
- Код, который хочет выполнить LLM
- Причину предупреждения (найденное опасное ключевое слово)
- Кнопки **Выполнить** и **Отменить**

Перед нажатием «Выполнить» — внимательно прочитайте код.
Если сомневаетесь — нажмите **Отменить**.

---

## Если 1С не отвечает

Когда MCP недоступен, вверху чата появляется красная плашка:
**«Подключение к 1С потеряно. Повторить»**

Действия:
1. Проверьте что MCP Toolkit (EPF) запущен и сервер активен
2. Нажмите кнопку **Повторить** в баннере
3. Если не помогает — перезапустите EPF в 1С и нажмите Повторить снова

Пока баннер активен — поле ввода заблокировано.

## FAQ

**Куда сохраняются мои сообщения?**
В локальную SQLite-базу на сервере (файл `data/app.db`). Данные не отправляются
в облако. При развёртывании на вашем сервере — вы владеете данными.

**Кто видит мой API ключ LLM?**
Только ваш браузер и ваш backend. Ключ хранится в localStorage браузера,
передаётся только на ваш собственный backend при каждом /chat запросе,
тот форвардит его к LLM-провайдеру и не сохраняет.

**Что делать если LLM отвечает не то что ожидалось?**
Переформулируйте вопрос точнее. Если нужен конкретный объект — укажите его имя,
тип или ИНН. Или нажмите **+ Новый чат** и начните с чистого контекста.

**Как удалить сессию?**
В левой панели (история) наведите курсор на сессию → появится иконка корзины.
Нажмите — сессия и все её сообщения удалятся (CASCADE).

**Почему таблица обрезана (не все строки)?**
TableCard показывает максимум 1000 строк. Для полного результата
переформулируйте запрос с конкретными фильтрами, или уточните у аналитика
насчёт CSV-экспорта через execute_query с дополнительной обработкой.

**Почему кнопка «Загрузить ещё» в журнале недоступна?**
Кнопка активна только для новых карточек (созданных после версии 3 схемы БД).
Старые сессии из Phase 2 не имеют card_id, поэтому load-more недоступен.

**Тёмная тема — единственная?**
Да. Светлая тема — вне области MVP. Запланировано в v2.

**Как добавить несколько баз 1С?**
Добавьте несколько подключений в Настройки → Подключения (каждое с разным URL/портом).
Переключайтесь через Channel selector в шапке.
